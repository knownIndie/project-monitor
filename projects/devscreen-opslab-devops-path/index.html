Project split

Do not turn DevScreen into infrastructure practice at the cost of finishing its product.

DevScreen

The real product.

  • CI and build safety
  • Environment validation
  • Health and migrations
  • Logs, metrics, and traces
  • Rollback and incident practice

OpsLab

The disposable learning service.

  • TypeScript incident API
  • PostgreSQL and failure routes
  • Docker and Compose
  • Terraform and Cloud Run
  • Kubernetes, Argo CD, Kargo

Not now

Platform theatre.

  • Docker in DevScreen
  • DevScreen on Kubernetes
  • Microservices and Kafka
  • Service mesh or GKE
  • Flux before Argo CD works
Hard constraint: DevScreen gets no Dockerfile, Compose stack, Cloud Run deployment, or Kubernetes manifests for this plan.

Concrete DevScreen trace

Trace submit attempt -> load answers -> AI evaluation -> validate JSON -> save report. Record evaluation duration and failure reason without logging resume text, answers, prompts, tokens, or secrets.

Verified blockers: Biome currently fails on dropdown formatting and warns about an unused varchar import. The production build compiles, then fails because callbackURL is not a valid top-level sign-out option.

OpsLab scope

Small enough to build in a weekend, useful enough to expose real operational failures.

Build

  • Fastify or Express with TypeScript
  • PostgreSQL incidents table
  • Create, list, and resolve incident routes
  • /health/live and /health/ready
  • Unit and API integration tests

Break safely

  • /debug/slow?ms=1000
  • /debug/error
  • Wrong database hostname
  • Connection pool exhaustion
  • Bad image and readiness failure
Scope rule: no auth, AI, payments, uploads, chat, or design system. OpsLab is a machine for learning operations, not another startup.

12-week route

Each row names the repo, the artifact, and the failure you must diagnose.

Week 1

DevScreen: baseline and CI

Biome, type-check, tests, production build, PR workflow.

Ship: required CI checks.

Break: a type error blocks merge.

Week 2

DevScreen: health and logs

Typed env validation, liveness, database readiness, request IDs.

Ship: clear health routes.

Break: missing database URL fails clearly.

Week 3

OpsLab: tiny API

Incident CRUD, PostgreSQL, tests, health and debug routes.

Ship: one-command local setup.

Break: port and database failures.

Week 4

OpsLab: Docker

Multi-stage non-root image, Compose, health checks, shutdown.

Ship: API plus database.

Break: wrong host, port, and command.

Week 5

OpsLab: container CI

Test, build, scan, publish immutable commit-SHA images.

Ship: traceable registry image.

Break: failing test blocks publish.

Week 6

OpsLab: Terraform

GCP APIs, registry, IAM, secrets, budget, Cloud Run.

Ship: disposable environment.

Break: detect drift and bad IAM.

Week 7

OpsLab: Cloud Run

Keyless CI deploy, revisions, scaling, identity, rollback.

Ship: production-style revision.

Break: bad revision, then roll back.

Week 8

OpsLab: observability

OpenTelemetry, JSON logs, RED metrics, dashboard, alert.

Ship: correlated signals.

Break: slow route and pool exhaustion.

Week 9

DevScreen: evaluation telemetry

Trace AI evaluation, validation, save, and user-visible result.

Ship: latency and failure dashboard.

Break: invalid AI JSON.

Week 10

OpsLab: local Kubernetes

kind or k3d, probes, resources, secrets, Kustomize.

Ship: local cluster deployment.

Break: probe and resource failures.

Week 11

OpsLab: Argo CD

Separate GitOps repo, reconciliation, health, drift, rollback.

Ship: Git changes desired image.

Break: manual cluster drift.

Week 12

OpsLab: Kargo

Warehouse, Freight, dev, staging, production, verification.

Ship: controlled promotion.

Break: failed verification blocks prod.

Working rules

Keep product progress ahead of infrastructure theatre.

No Docker in DevScreen. Container learning stays in OpsLab.
No blind copying. Read, close the docs, implement, then compare.
Break each phase. The failure diagnosis is part of the deliverable.
Protect user data. Never log answers, resumes, prompts, tokens, or secrets.
Keep evidence. Save CI runs, plans, traces, dashboards, and incident notes.
One GitOps path. Learn Argo CD with Kargo before comparing Flux.
